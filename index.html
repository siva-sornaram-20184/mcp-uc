What Is MCP?

Model Context Protocol — a standardized way for AI models to call security tools in real-time.

How It Works

  • AI client receives analyst's question in natural language
  • Each security vendor publishes its own MCP server independently
  • AI client connects to multiple vendor MCP servers simultaneously
  • AI orchestrates cross-vendor queries, correlating results in real-time
  • Results synthesized into a single, actionable narrative

What Is Zoho MCP?

Zoho MCP is a product built around the Model Context Protocol — enabling AI agents to interact with 55+ Zoho & ManageEngine apps and 500+ third-party integrations through a standardized interface. For security teams, this means tools like Endpoint Central and ServiceDesk Plus become AI-accessible out of the box.

Key Capabilities

  • Agent-ready apps: AI agents can execute actions across Zoho & ManageEngine products via natural language — from CRM to endpoint security
  • Works with all LLMs: Compatible with Claude, GPT, and other AI agents out of the box
  • 500+ integrations: Connects Zoho apps with third-party services (Slack, GitHub, Stripe, Twilio, etc.)
  • API-native execution: Translates natural language intent into real API calls — e.g., patch scans, vulnerability reports, ticket creation

What happened: A Kerbrute-style password spray from internal IP 192.168.1.100 targeted 11 accounts on domain controller DC01 within a 3-minute window (09:15–09:18 UTC, 12 Mar).

Compromised account: finance1 — attacker obtained a valid TGT (Event 4768, Result 0x0), then used RDP to pivot to workstation WS01 for reconnaissance (net user, net group commands observed).

EDR response: CrowdStrike flagged 3 detections and blocked the Domain Admin credential abuse on WS01 before further lateral movement could occur.

MITRE mapping: T1110.003 (Spray) → T1078 (Valid Accounts) → T1021.001 (RDP) → T1087 (Discovery) → T1078.002 (Domain Accounts) — full chain covered across Log360 + CrowdStrike.

Overall risk is CRITICAL — active attacker reached Domain Admin. EDR contained the abuse, but full remediation (password resets, source host forensics, lateral movement audit) is required.
